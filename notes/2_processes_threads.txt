// i gets assigned memory addr, say 1500.
// STORE [1500], 0
int i = 0;

// LOAD Ax, [1500] -> opcode 01101111 | Ax register 0000 | mem addr
// LOAD Bx, 10
// ADD Ax, Ax, Bx
// STORE [1500], Ax
i = i + 10;

Program execution
* C program on HDD -compiler-> assembly -assembler-> binary executable
* executable gets loaded into RAM, starts getting executed line by line

* PC (program counter) points to addr of next instruction to run
* instruction is read into register, then executed

program switching (context switch)
* multiple processes running, CPU must switch
* OS runs timer in background, interrupt suspends process, runs OS
* Proc1 | OS | Proc2 
* switch: PC gets set to OS code addr
* if switch on interrupt (BAD): P1 stuff in registers gets overwritten

on interrupt
* hardware dumps process state (next PC, all registers)
* OS saves process state
* to give back control OS->P1: reload P1 state

process creation: fork

process terminitation
* normal exit (voluntary)
* error exit (voluntary)
* fatal error (involuntary), seg fault, run out of stack memory
* killed by other process (involuntary)

process states
* running
* blocked: I/O, syscall to read file
* ready
* OS has process table, runs ready procs

* P1 wants to read file (syscall), gives up control (dump state)
* OS marks P1 blocked, schedules P2
* HDD raises interrupt, tells OS file is ready
* OS marks P1 ready

scheduler: picks proc to execute

process table entry (OS managed)
* PID (process ID)
* state: running, blocked, ready
* dumped state: all registers, PC
* files opened
* pointer to process memory

program memory
* stack: function call (stack frames), grows down
* heap: allocated memory (malloc + free), grows up
* data: global/static 
* code

hardware interrupt
* highest priority interrupt code is output
* Interrupt Vector Table: code -> interrupt instruction addr
* instruction addr is loaded into PC, interrupt executes

interrupt
* hardware dumps PS, registers
* hardware loads new PC from interrupt vector
* asm saves dumped registers
* asm sets up stack for interrupt
* interrupt runs
* scheduler runs next proc
* asm prepares next proc, finally loads PC

CPU utilization = 1 - (IO wait percent)^procs
* 2 procs, 20% IO wait, P(both blocked by IO) = (0.2)^2 = 0.04, CPU util = 1 - 0.04 = 0.96
* more procs = higher CPU util

1024 KB RAM: 256 KB OS, 100 KB per Proc, 30% IO wait
* space for procs = 1024 - 256 = 768
* max 7 Procs
* CPU util = 1 - 0.3^7
can use swap space in HDD for RAM for more procs
