\section{Ch 3: Memory}

\begin{enumerate}
    \item Local vs global alloc

        local: page replaces page in same process (working set is local b/c uses local process time)

        global: page replaces page in any process

    \item space per proc: equal alloc, proc size

        page fault rate per proc, boundary for page fault rate, high page fault rate = get more pages

        load control: all procs need more pages, suspend some and give up pages, mix CPU-bound + IO-bound

    \item page table overhead = (process size / page size) * page entry size + (page size / 2)

        optimal page size = sqrt(2 * process size * page entry size)

    \item separate instruction and data spaces

        same program running multiple times: instruction space is the same, data space is copied on write

        shared libraries (dll + so): programs have pointer to library in ram (don't have to load multiple times, update at once)

    \item cleaning policy

        paging daemon: periodically evict + write pages if available pages is too low

        2 handed clock: 1st hand = cleaning daemon, 2nd hand = page replacement alg

    \item page fault: hardware interrupt (not in TLB), kernel checks page table + maybe fetch page

    \item implementation issues

        \begin{enumerate}
            \item instruction backup: page fault vauses instruction to stop halfway through,
                OS doesn't know where to restart


            \item A does IO + gets suspended, B gets page fault (global page replacemnet), claims file A is using for IO

                pinning: lock page

                kernel buffers for IO
        \end{enumerate}

    \item backing store: where to put evicted pages in disk

        static swap: swap area same size as virtual addr space, stores pages not in page table

        dynamic: swap area is smaller, disk map

    \item segmentation: each variable can get own segment (data, stack, heap), programmer (compiler) defines segments

        without paging: whole segments get brought in / swapped out, compaction (address translation w/ + base register), physical addr = segment table[segment number] + offset

        with paging: MULTICS
        \begin{enumerate}
            \item descriptor segment has pointers to page tables
            \item multics virtual address = segment number + page number + offset
            \item physical addr = segment table [segment number] [page number] + offset
            \item multics tlb: index on segment number, virtual addr, gets page number
        \end{enumerate}

        Pentum: 256K segments, 64k words
        \begin{enumerate}
            \item selector: global (0) (for OS / system) vs local (1) descriptor table (for program), index
            \item (global / local) descriptor table [index] = selector, offset
            \item selector points to segment descriptor, linear addr = segment base + offset

                with paging: linear addr = virtual addr = page dir number | page table number | offset
        \end{enumerate}
\end{enumerate}


\section{Ch 6: Deadlocks}
\begin{enumerate}
    \item deadlock causes
        \begin{enumerate}
            \item resource requires mutual exclusion
            \item processes can request new resources
            \item processes cannot be preempted
            \item circular wait chain

                solution: number resources, you can only acquire in order
        \end{enumerate}

    \item detection: cycles are deadlocks, matrix method
    \item recovery: preempt, rollback, kill
    \item dynamic avoidance: check at resource allocation time
    \item 2 phase locking: lock before changing, databases

        communication deadlock: timeout + sequence numbering

        livelock: dining philosophers

        priority starvation
\end{enumerate}


\section{Ch. 4: Filesystems}
\begin{enumerate}
    \item file system layout
        \begin{enumerate}
            \item boot sequence: bios, mbr, boot block, loads os
            \item master boot record: loads active partition
            \item partitions: boot block ...
        \end{enumerate}

    \item allocation

        contiguous

        linked list alloc: pointer to next, block. no random access, must be sequential

        FAT (file allocation table): idx of next block, linked list in table

        INodes (linux): file attr + addr to data blocks + addr to (1x, 2x, 3x) indirect blocks
        \begin{enumerate}
            \item indirect block: pointer to block of pointers to data blocks
            \item symlink (file w/ path) vs hard link (also in inode)
            \item get data block for /root/home/me.txt:
                \begin{enumerate}
                    \item root inode in memory. find inode for home.
                    \item fetch home inode. fetch home block.
                    \item find me.txt inode. fetch me.text block.
                \end{enumerate}
        \end{enumerate}

    \item backups

        physical dump: dump everything

        logical dump: bitmap for modified files, only dump chains of dirs w/ modified files
\end{enumerate}
